# Retired Paths

This file records paths that should not be revived without new evidence and, when appropriate, an ADR.

| Path/name | Status | Why retired | Replacement | Revisit condition |
| --- | --- | --- | --- | --- |
| External LLM/Cortex/ThoughtLoop runtime path | retired | Added external dependency and ambiguous cognition claims without being the living substrate. | Subcortex-owned runtime evidence, semantics/readout surfaces, replay, and Runtime Truth. | A future ADR proves a bounded dependency does not become the cognition substrate and preserves evidence gates. |
| Manager mixin compatibility aliases | retired | Preserved shallow ownership after the service split. | Explicit deep modules and RuntimeFacade/StatusReadModel ownership. | Only if a compatibility period is required by a released public API. |
| Static CUDA intent as capability proof | retired | Configuration can hide CPU fallback. | Observed tensor/device evidence in Runtime Truth and gates. | Never as proof; static intent may remain diagnostic only. |
| Local NVIDIA NIM/Cortex dotenv configuration | retired | The external LLM/Cortex adapters and embedder are deleted, so keeping local NIM credentials implies a capability the runtime cannot and must not use. | Local Hugging Face source credentials only; MARULHO-owned Subcortex/SNN cognition remains local. | Only through a new ADR that explicitly reopens an external non-cognition source/client boundary. |
| Routing-index reconstruction-distance reuse with pending overlay | rejected experiment | Exact numerical parity was possible, but the complete 1024-column CUDA routing pipeline regressed by 20.6% median and 22.3% p95 because expanded search and overlay control cost exceeded the removed dense scan. | Keep the fresh dense reconstruction scan and the separately proven candidate-scoped competitive scoring path. | Revisit only as a fused device-side pipeline benchmarked against the complete routing step. |
| Raw Wikipedia as default Terminus background source | retired | HF source review found a drop-in open-textbook stream with direct `text` rows and denser instructional/worked-example content for continuous background learning. Raw full-text Wikipedia remains useful reference material but is no longer the best default stream. | `open_textbooks` (`izumi-lab/open-text-books`) plus S2ORC ArXiv abstracts and FineWeb-Edu in the explicit Terminus Source Bank. | Revisit through structured Wikipedia only after Runtime Sources owns a section/table-aware adapter and bounded sampling policy. |
| Hot-path dead-column auto-revival | retired | Routine competitive-column processing scanned all stale counters and could structurally revive columns during every tick, adding CUDA synchronization/scan cost and hiding structural mutation inside routing. | Hot-path ticks record stale counters and spike-health evidence only; explicit deep-sleep/maintenance paths call `force_revive_dead_columns` when revival is warranted. | Revisit only with a fused/event-driven maintenance mechanism that beats the current benchmark and preserves reversible mutation evidence. |
| Hot-path hypercube hub topology refresh | retired | Every `HypercubeBindingLayer.bind()` updated hub evidence and immediately refreshed structural outreach, allowing adjacency growth/pruning and mutation-ledger writes during ordinary ticks without checkpoint/operator authorization. | Bind updates activation EMA only. The explicit core refresh is reachable solely through the checkpoint-backed, revision-bound, operator-confirmed Structural Mutation Application, with target/reason hash binding, edge-budget enforcement, exact delta evidence, and rollback. | Never restore per-bind refresh. Further automation requires a separate evidence-backed promotion decision while preserving the same transaction guarantees. |
| Always-running inactive Hypercube Binding | retired | A representative 1024-column CUDA checkpoint had zero binding usage and output, but the binding specialist still launched its full local computation every token. This violated the column-society rule that dormant specialists should not consume routine metabolism. | Conditional Binding Wake probes inactive binding at a bounded cadence, preserves cached state between probes, exposes execution evidence, and immediately resumes every-tick execution after learned or restored usage becomes active. | Revisit the cadence only with prediction/learning-quality evidence and an end-to-end benchmark; do not restore unconditional inactive execution as compatibility behavior. |
| Always-running text-only Cross-Modal Grounding | retired | Live profiling showed `on_text_spike()` cost about `3.9 ms` median every token even when no visual/audio evidence was present, so text-only background ticks spent grounding metabolism without a sensory co-occurrence event. | Conditional Cross-Modal Text Wake runs text-only grounding at a bounded cadence, records cached-idle skips, preserves sensory-backed every-tick updates, and exposes Runtime Truth counters. | Revisit only with grounding-quality evidence plus endpoint and p95 benchmarks; do not restore unconditional text-only grounding in the hot path. |
| Zero-blend lightweight input-synapse rewrites | retired | With `input_weight_blend=0.0`, lightweight input weights cannot affect competition or assembly output, but the hot path still rewrote and renormalized the winner row every tick. Controlled RTX 3060 hot-window probes showed lower median and p95 latency when this zero-contribution work was skipped. | Keep prototype learning, homeostasis, stale counters, and spike evidence active; update lightweight input synapses only when their blend is nonzero. Local STDP remains active because it owns additional projection, inhibition, scaling, and spike-state mechanisms. | Revisit only by enabling a nonzero blend or through a separately benchmarked plasticity mechanism whose state contributes to live cognition. |
| ConceptStore refresh from binding-topology structural evidence | retired | The structural design and isolated evaluation measured binding edge deltas, but the executor called `ConceptStore.refresh_structural_capacity`, so review evidence did not identify the state being mutated. | Structural Mutation Application now targets only `HypercubeBindingLayer.refresh_hub_topology`; concept-capacity mutation needs its own evidence and transaction if promoted later. | Reintroduce only as a separately named target with target-specific evaluation, checkpoint verification, bounded delta, and rollback tests. |
| Row-wise CUDA scalar scan for binding-growth planning | rejected experiment | Reading sparse degree/neighbor rows one scalar at a time synchronized the host repeatedly and measured about `303.2645 ms` median at 1024 columns. | Snapshot adjacency tensors twice, then perform vectorized candidate-membership planning on the CPU control plane; measured CUDA-backed median fell to `5.1334 ms`. | Revisit only as a fused device-side sparse planner that beats the bounded snapshot path and preserves exact plan hashes. |
| External source `close()` under the manager lock | retired | A blocked Hugging Face iterator shutdown could hold the shared `RLock`, freezing stop, status, and checkpoint operations together. | Stop sets the cooperative event and records lifecycle state without invoking external stream shutdown under the manager lock; checkpoint save fails fast while runtime mutation is active. | Revisit only with a proven bounded, thread-safe cancellation primitive that cannot execute provider I/O while holding runtime state locks. |
| SNN language thought-era public HTTP route vocabulary | retired | Public readout-ledger routes now use canonical `snn-language-*` route names and `snn_language_*` request fields. Keeping the previous route families documented individually made old vocabulary look more important than the current API. | Canonical SNN language surface, memory, consolidation, structural plasticity, capacity mutation, newborn-neuron integration, critical-period learning, maturation review, and newborn-synapse pruning routes over the same reviewed ledger/executor gates. | Only if a real external integration appears; this local system does not keep compatibility routes by default. |
| Legacy multi-tab dashboard and decorative neural-space UI | retired | Thirteen overlapping monitor/model/control tabs, duplicated API state, and Three.js/R3F neural animation preserved the old system vocabulary while increasing bundle weight and implying runtime activity without Runtime Truth evidence. | Six maintained operator workspaces using shadcn controls, plus a read-only 2D Runtime Machinery Plane driven by current source, device, sparse-column, vote, memory, binding, and execution telemetry. | Reintroduce a visualization only when it maps a maintained Runtime Truth field or operator workflow and its bundle/runtime cost is measured. |
| Functional full-state compiled steady-state column transition | rejected experiment | Isolated `torch.compile(mode="reduce-overhead")` execution improved substantially, but the functional transition returned eleven full state tensors and copied them into stable runtime buffers. Two controlled full hot-window A/B pairs averaged about `34.75` versus `36.65 tokens/sec` for compiled versus the retained live path, with mixed median/p95 latency and about `14 s` first-use warmup. | Keep the parity function and runner as evaluation-only evidence. The production target is an in-place CUDA/Triton or persistent-graph kernel that mutates runtime-owned buffers directly and wins repeated full-`train_step` A/B tests. | Revisit only when the implementation avoids full-state functional outputs, preserves checkpoint/parity semantics, reports device and memory evidence, and consistently improves complete sequential cognitive ticks. |
